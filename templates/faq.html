{% trans "What is IssueTracker?" %}

{% blocktrans %}IssueTracker is a very simple issue tracking application written using the Django web framework and a bespoke workflow engine. {% endblocktrans %}

{% trans "Who wrote IssueTracker?" %}

{% blocktrans %}Nicholas Tollervey wrote this software over a period of two days as an academic exercise. {% endblocktrans %}

{% trans "Why write yet another issue tracker?" %}

{% blocktrans %}Like I just said, as an academic exercise. My aim being to use the workflow engine "in anger" so I get a feel for how helpful it is to code against. Hopefully, the pain I encountered when writing this will inform me when updating the project. {% endblocktrans %}

{% trans "How much does it cost?" %}

{% blocktrans %}Nothing. :-) {% endblocktrans %}

{% trans "What about the workflow engine?" %}

{% blocktrans %}The workflow engine is a generic application that I can "drop in" to any Django based web-application. The only thing needed to finish it off is a set of forms and UI views to make creating and editing workflow easier (rather than having to use the Django admin area). {% endblocktrans %}
